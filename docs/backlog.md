## Автоматизация и алертинг

### Механизм алертинга (референс: perflog модуль)
Модуль `perflog` из `bx24.stat` отправляет алерты в чат Bitrix24 через webhook:
```
POST {webhook_url}/im.message.add.json
{
  "DIALOG_ID": "chatXXXX",
  "MESSAGE": "[B]Alert[/B]\nLCP degraded...",
  "SYSTEM": "Y"
}
```
Формат сообщений — BB-code (поддержка `[B]`, `[URL]`, `[CODE]`).
Настройки: webhook URL + chat ID в конфиге.

### Уровень 1 — Scheduled runs
- [ ] Cron-скрипт: `node scripts/scheduled-run.ts --profile <id>` → создаёт run, стартует, ждёт завершения
- [ ] Или встроенный scheduler в API (setInterval / node-cron)
- [ ] История метрик: после каждого прогона сохранять ключевые метрики (LCP, total JS, request count) в отдельную таблицу/файл для трендов

### Уровень 2 — Thresholds + алерты в Bitrix24 чат
- [ ] Настройки в UI: webhook URL, chat ID, правила (JSON)
- [ ] Правила:
  - `total_js_decoded > 3MB` → warn
  - `lcp > 4000` → critical
  - `vue2_detected` → alert
  - `new_asset_size > 500KB` → notify
  - `asset_count_delta > 20` → warn (по сравнению с предыдущим прогоном)
- [ ] После каждого прогона: проверить правила → если сработали → отправить в чат
- [ ] Формат сообщения: BB-code, ссылка на прогон в WebPerf Hub
- [ ] Дедупликация: не слать повторно если проблема уже была в предыдущем прогоне

### Уровень 3 — Trend detection
- [ ] Хранить метрики за N прогонов, строить тренды
- [ ] Алерт на деградацию: «LCP вырос на 30% за неделю», «Total JS растёт 5 прогонов подряд»
- [ ] Графики трендов в UI (sparklines или полноценные чарты)
